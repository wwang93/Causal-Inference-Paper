  • Show All Code

Overview

This appendix illustrates a simple two-period DiD setup using HSLS variables. The outcome is science self-efficacy measured at baseline (X1SCIEFF) and follow-up (X2SCIEFF). Schools are treated if they adopt a PD requirement by follow-up (while having no requirement at baseline). We estimate a baseline DiD and then a version that adds student covariates for precision.

A3.7 Define treated vs never-treated schools (based on adoption)

We restrict to schools with no requirement at baseline (a1 == 0).

Then:

  • treated schools: adopt by follow-up (a2 == 1)
  • control schools: never adopt (a2 == 0)

hsls_did <- analytic_clean %>%
  filter(a1 == 0) %>%
  mutate(
    treat_school  = ifelse(a1 == 0 & a2 == 1, 1, 0),
    never_treated = ifelse(a1 == 0 & a2 == 0, 1, 0)
  ) %>%
  filter(treat_school == 1 | never_treated == 1)

A3.8 Simulate school IDs for clustering (SCH_ID_SIM)

In a proper DiD, you normally already have a stable school identifier for clustering.

Here we simulate a school ID assignment to keep the example self-contained (and to show the mechanics of clustering in fixest).

Code 
# Student counts by adoption status at follow-up
n_treatment_students <- sum(hsls_did$a2 == 1, na.rm = TRUE)
n_control_students   <- sum(hsls_did$a2 == 0, na.rm = TRUE)

# Allocate schools proportionally (tweak if you want)
n_total_schools <- 80
prop_treatment <- n_treatment_students / (n_treatment_students + n_control_students)

n_schools_treatment <- round(n_total_schools * prop_treatment)
n_schools_control   <- n_total_schools - n_schools_treatment

# Assign simulated school IDs
hsls_did$SCH_ID_SIM <- NA_integer_

idx_treatment <- hsls_did$a2 == 1
hsls_did$SCH_ID_SIM[idx_treatment] <-
  sample(1:n_schools_treatment, sum(idx_treatment), replace = TRUE)

idx_control <- hsls_did$a2 == 0
hsls_did$SCH_ID_SIM[idx_control] <-
  sample((n_schools_treatment + 1):(n_schools_treatment + n_schools_control),
         sum(idx_control), replace = TRUE)

A3.9 Reshape to long format (two rows per student)

We stack baseline and follow-up outcomes, then create:

  • post: 0 baseline, 1 follow-up
  • treat_post: DiD interaction term
Code 
cols_to_keep <- c(
  "STU_ID", "SCH_ID_SIM", "treat_school",
  "X1SCIEFF", "X2SCIEFF",
  "female", "race_white", "race_black", "race_asian",
  "dual_lang", "locale_rural", "SES"
)

hsls_long <- hsls_did[, cols_to_keep] %>%
  pivot_longer(
    cols      = c(X1SCIEFF, X2SCIEFF),
    names_to  = "wave",
    values_to = "SCI_SE"
  ) %>%
  mutate(
    post       = ifelse(wave == "X2SCIEFF", 1, 0),
    treat_post = treat_school * post
  )

table(hsls_long$post)

   0    1 
6356 6356

A3.11 Mean trends plot (treated vs control)

This plot is a quick visual check of how treated and control groups move from baseline to follow-up.

With only two time points, we can’t really “test” parallel trends, but at least we can see whether the groups are moving in obviously different ways.

Code 
trend_dat <- hsls_long %>%
  group_by(treat_school, post) %>%
  summarise(
    mean_SCI_SE = mean(SCI_SE, na.rm = TRUE),
    .groups = "drop"
  )

ggplot(
  trend_dat,
  aes(
    x = post,
    y = mean_SCI_SE,
    group = factor(treat_school),
    linetype = factor(treat_school),
    shape = factor(treat_school)
  )
) +
  geom_line() +
  geom_point(size = 3) +
  geom_vline(xintercept = 0.5, linetype = "dashed", color = "gray50", linewidth = 0.8) +
  scale_x_continuous(breaks = c(0, 1), labels = c("Baseline", "Follow-up")) +
  labs(
    x = "Wave",
    y = "Mean science self-efficacy",
    linetype = "Treated school",
    shape = "Treated school",
    caption = "Dashed line marks the intervention period."
  ) +
  theme_minimal()

A3.12 Sensitivity analysis (Konfound)

Below we run konfound using the estimate and SE you provided for the DiD effect.

Code 
pkonfound(
  est_eff      = -0.0346,
  std_err      = 0.1025,
  n_obs        = 6356,
  n_covariates = 2,
  index        = "RIR"
)
Robustness of Inference to Replacement (RIR):
RIR = 5262

The estimated effect is -0.035. The threshold value for statistical significance
is -0.201 (with null hypothesis = 0 and alpha = 0.05). To reach that threshold,
82.78% of the estimate of -0.035 would have to be due to bias. This implies to sustain
an inference one would expect to have to replace 5262 (82.78%) observations with
effect of 0 with data points with effect of -0.201 (RIR = 5262).

See Frank et al. (2013) for a description of the method.

Citation: Frank, K.A., Maroulis, S., Duong, M., and Kelcey, B. (2013).
What would it take to change an inference?
Using Rubin's causal model to interpret the robustness of causal inferences.
Education, Evaluation and Policy Analysis, 35 437-460.

Accuracy of results increases with the number of decimals reported.
Code 
pkonfound(
  est_eff      = -0.0346,
  std_err      = 0.1025,
  n_obs        = 6356,
  n_covariates = 2,
  index        = "IT"
)
Impact Threshold for a Confounding Variable (ITCV):
The maximum impact of an omitted variable to sustain an inference for a null hypothesis
of an effect of 0 (nu) is based on a correlation of 0.141 with the outcome and 0.141
with the predictor of interest (conditioning on all observed covariates in the model;
signs are interchangeable if they are different). This is based on a threshold effect of
-0.201 for statistical significance (alpha = 0.05).

Correspondingly the maximum impact of an omitted variable (as defined in Frank 2000) is 
0.141 X 0.141 = 0.02 to sustain an inference for a null hypothesis of an effect of 0 (nu).

For calculation of unconditional ITCV using pkonfound(), additionally include
the R2, sdx, and sdy as input.

See Frank (2000) for a description of the method.

Citation:
Frank, K. (2000). Impact of a confounding variable on the inference of a
regression coefficient. Sociological Methods and Research, 29 (2), 147-194

Accuracy of results increases with the number of decimals reported.

The ITCV analysis was originally derived for OLS standard errors. If the
standard errors reported in the table were not based on OLS, some caution
should be used to interpret the ITCV.

```{r}
#| label: a3-long-format
#| include: true

cols_to_keep <- c(
  "STU_ID", "SCH_ID_SIM", "treat_school",
  "X1SCIEFF", "X2SCIEFF",
  "female", "race_white", "race_black", "race_asian",
  "dual_lang", "locale_rural", "SES"
)

hsls_long <- hsls_did[, cols_to_keep] %>%
  pivot_longer(
    cols      = c(X1SCIEFF, X2SCIEFF),
    names_to  = "wave",
    values_to = "SCI_SE"
  ) %>%
  mutate(
    post       = ifelse(wave == "X2SCIEFF", 1, 0),
    treat_post = treat_school * post
  )

table(hsls_long$post)
```
## A3.10 DiD estimation (two-way setup with school fixed effects)

### (a) Basic DiD

This is the clean baseline model: outcome on `post` and `treat_post`, with school fixed effects.
```{r}
#| label: a3-did-basic
#| include: true

did_basic <- feols(
  SCI_SE ~ post + treat_post | SCH_ID_SIM,
  data = hsls_long
)

summary(did_basic)
```

### (b) DiD with student covariates (precision controls)

Adding covariates is not required for identification in DiD, but it can help precision and guard against small sample quirks.
```{r}
#| label: a3-did-covariates
#| include: true

did_cov <- feols(
  SCI_SE ~ post + treat_post +
    female + race_white + race_black + race_asian +
    dual_lang + locale_rural + SES | SCH_ID_SIM,
  data = hsls_long
)

summary(did_cov)
```
